# Onboarding a New Developer

**Give them one doc:** [specs/PHOENIX_V4_CANONICAL_SPEC.md](specs/PHOENIX_V4_CANONICAL_SPEC.md)

That’s the full spec — content model (6 section types, atom schema), structure (tiers, formats, beat maps), assembly (plan compiler, K-tables), governance (approval, CI), and directory layout. No other spec overrides it.

**Optional:** Originals of earlier docx specs are in `archive/specs_originals/` if they need to check something from the source exports.
