Creating effective thumbnails for self-help content on YouTube (10-minute videos), Instagram, and TikTok requires eye-catching visuals that promise transformation and emotional resonance. Focus on simplicity, faces, bold text, and high contrast to boost click-through rates across platforms.

## Core Principles
Use human faces showing strong emotions like determination or "aha" moments to draw clicks—thumbnails with faces outperform others by triggering curiosity. Keep designs minimal: one focal image, 3-6 words of punchy text (e.g., "5 Habits to 10X Life"), and vibrant colors like reds/oranges for energy. Ensure high-resolution (1280x720 for YouTube; square 1080x1080 for IG/TikTok) and test variations via platform analytics. [natashalanedesignco](https://natashalanedesignco.com/tips-for-creating-captivating-youtube-thumbnails/)

## Platform-Specific Tips
**YouTube (10-min videos):** Pair a close-up face with numbered hooks like "Change in 10 Min" overlayed in large, sans-serif font; apply rule of thirds for balance. Brand consistently with your logo or color scheme for recognition. [support.google](https://support.google.com/youtube/answer/12340300?hl=en)

**Instagram Reels:** Go vertical (1080x1920), emphasize text-first for quick scrolls (e.g., "End Anxiety NOW"), and add subtle gradients for motivational vibes. [youtube](https://www.youtube.com/watch?v=Jb8zqH5u4Dw)

**TikTok Shorts:** Bold, trendy aesthetics like dark moody backgrounds with glowing text; frequent image changes in video previews help, but static thumbs need instant impact. [tiktok](https://www.tiktok.com/@themuthership/video/7220915298629651755)

## Tools and Workflow
Leverage free tools like Canva templates for self-help thumbnails (search "motivational") or Photoshop for custom AI-generated faces via Midjourney prompts like "inspirational portrait, intense gaze, warm lighting." Batch-create 3-5 versions per video, A/B test on YouTube, and iterate based on CTR—aim for 5-10%. For automation, Python scripts with Pillow library can generate variants from templates, aligning with your coding workflow. [thumbnailtest](https://thumbnailtest.com/guides/best-practices-youtube-thumbnail/)

A/B testing thumbnails boosts click-through rates (CTR) for self-help videos on YouTube and TikTok by pitting design variations against each other to find winners based on real viewer data. YouTube offers a native tool, while TikTok relies on manual or third-party methods—test 2-3 distinct versions (e.g., face vs. text-heavy) per video for best results. [tubebuddy](https://www.tubebuddy.com/tools/youtube-thumbnail-test)

## YouTube Testing
Use YouTube's free "Test & Compare" feature in Studio (desktop): Select a video, upload 2-3 thumbnails, set test duration (hours to 14 days or until 95% statistical significance), and YouTube randomizes them to viewers while tracking CTR/views. Results auto-apply the winner or report ties; test older videos first to avoid early momentum dips, and make variants bold (e.g., **Roboto** text swaps like "End Anxiety" vs. "10X Mindset"). Tools like **TubeBuddy** add title/tag testing with similar steps. [epidemicsound](https://www.epidemicsound.com/blog/a-b-test-youtube-thumbnails/)

## TikTok Testing
No built-in thumbnail A/B tool exists, so upload variations as separate Shorts, promote via ads (split-test audiences), or track analytics manually over 24-48 hours for swipe-up rates. Alternate thumbs on reposts or use third-party schedulers like Later for audience splitting; focus on vertical formats with **Poppins** bold hooks, as feed velocity favors instant readability. [tiktok](https://www.tiktok.com/@milostrategy/video/7299187853043551520)

## Best Practices
Run tests on high-impression videos (1K+ views) for reliable data; aim for 5-10% CTR lift by iterating weekly. Avoid similar variants (change colors/emotions) to speed results, and apply winners channel-wide for self-help consistency. [youtube](https://www.youtube.com/watch?v=U0S-TuEaLwE)

For bold, readable text on mobile thumbnails—especially self-help content on YouTube, IG, and TikTok—prioritize sans-serif fonts with high x-height, generous stroke weight, and tight kerning for quick glances during scrolls. These ensure legibility at small sizes (e.g., 20-60pt) across iOS/Android screens without pixelation or crowding. [fontsarena](https://fontsarena.com/blog/best-font-for-mobile/)

## Top Fonts
- **Roboto** Bold: Android default, clean geometry for sharp edges on any DPI. [fontsarena](https://fontsarena.com/blog/best-font-for-mobile/)
- **San Francisco** (SF Pro/SF Compact): iOS native, optimized for Retina with dynamic thickness. [din-studio](https://din-studio.com/5-best-fonts-for-mobile-apps/)
- **Open Sans** ExtraBold: Neutral, open counters prevent blurring at distance. [ironmarkusa](https://ironmarkusa.com/best-fonts-for-digital-media/)
- **Montserrat** Bold: Geometric with strong contrast, ideal for motivational hooks. [fontsarena](https://fontsarena.com/blog/best-font-for-mobile/)
- **Poppins** Bold: Rounded yet firm, versatile for vertical mobile formats. [fontsarena](https://fontsarena.com/blog/best-font-for-mobile/)

## Usage Tips
Limit to 1-2 fonts per thumbnail: Bold for headlines (e.g., "10X Your Mindset"), pair with medium body if needed. Use 200-300% stroke weight scaling in Canva/Photoshop, white/yellow text on dark gradients for 8:1 contrast ratio, and test at 30% zoom. Free via Google Fonts; avoid thin serifs or scripts—they fail on motion-heavy feeds. [craftsupply](https://craftsupply.co/articles/best-bold-fonts/)

## 1. **LLM step: understand the script, then define the image**

**Idea:** Before making image prompts, an LLM reads each segment, understands what’s being said, and returns **metadata** used to prompt the image (so we “listen,” then “pull the picture”).

**New script: `run_segment_scene_extraction.py`**
- **Input:** `script_segments.json` (your 7 sections).
- **Per segment:** Sends the segment text (+ topic, segment_id, slot) to an **OpenAI-compatible chat API** (so you can use Enlightened Intelligence V2 or any JSON-capable LLM).
- **Prompt:** Asks the model to output JSON with:
  - **scene_description** — one short sentence for the image, **specific to the script** (no generic “hands” or abstract symbols unless the script says so).
  - **mood** — one word.
  - **avoid** — a few things to avoid in the image.
- **Output:** `segment_scenes.json` (one such record per segment).

**Config:** `config/video/segment_scene_llm.yaml`  
- System hint and output-schema hint for the LLM.  
- Env: `OPENAI_BASE_URL`, `OPENAI_API_KEY`, optional `OPENAI_MODEL` (default `qwen2.5:latest`). Point the base URL at your EI V2 (or other) endpoint.

---

## 2. **Per-segment image generation from that metadata**

**New script: `run_flux_per_segment_build.py`**
- **Input:** `segment_scenes.json` (from step 1).
- **Per segment:** Uses `scene_description` as the scene input to your existing FLUX prompt builder (`get_prompt_for_topic_scene(topic, scene_description)`), then calls FLUX once per segment.
- **Output:**
  - `segment_images/<plan_id>_seg_1.png` … `<plan_id>_seg_7.png`
  - `segment_asset_index.json`: maps `segment_id` → `asset_id` for the resolver.

So prompts are driven by **script-specific** scene text from the LLM, not by a single generic “hands”/“symbol” scene.

---

## 3. **Pipeline wiring**

- **Asset resolver** accepts optional `--segment-index` (path to `segment_asset_index.json`). When provided, each shot is resolved by **segment_id** to the corresponding segment asset (no intent-based bank lookup for those shots).
- **Run pipeline** (`run_pipeline.py`):
  - New flag: `--segment-index` (optional).
  - If you don’t pass it but `out_dir/segment_asset_index.json` exists (e.g. after running the segment flow into that plan dir), the pipeline uses it automatically.
  - When a segment index is used, render step uses `plan_dir/segment_images` as the assets dir if you don’t pass `--assets-dir`.

So: **LLM understands the script → outputs metadata → we build one prompt per segment from that metadata → one image per segment → pipeline uses those images by segment.**
